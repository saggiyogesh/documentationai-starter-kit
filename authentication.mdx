## Overview

Secure your Yogesh Agrawal documentation space with robust authentication and granular permissions. You control access through user logins, API keys, and role-based access control (RBAC). This ensures only authorized users can view, edit, or manage your project docs.

Yogesh Agrawal supports multiple authentication methods to fit your workflow. Assign roles like Viewer, Editor, or Admin to team members. Invite collaborators with specific access levels to collaborate securely.

<Callout kind="alert">
Never share your API keys or passwords in public repositories or client-side code. Always use environment variables.
</Callout>

## Authentication Methods

Choose from several authentication options. Use the method that best suits your integration needs.

<Tabs>
  <Tab title="Email/Password" icon="mail">

    Authenticate users via email and password. This method uses a secure login endpoint.

    <Request tabs="JavaScript,cURL" show-lines="true">
      ```javascript
      const response = await fetch('https://api.example.com/v1/auth/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          email: 'user@example.com',
          password: 'your-password-here'
        })
      });
      const data = await response.json();
      ```
      ```bash
      curl -X POST https://api.example.com/v1/auth/login \
        -H "Content-Type: application/json" \
        -d '{"email":"user@example.com","password":"your-password-here"}'
      ```
    </Request>

    <Response tabs="200">
      ```json
      {
        "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "user": {
          "id": "user_123",
          "email": "user@example.com",
          "role": "editor"
        }
      }
      ```
    </Response>

  </Tab>
  <Tab title="API Keys" icon="key">

    API keys provide token-based access for integrations and scripts.

    <ParamField header="X-API-Key" param-type="string" required="true">
      Your secret API key. Generate one from your dashboard at `https://dashboard.example.com/api-keys`.
    </ParamField>

    <ParamField header="X-Client-ID" param-type="string" required="true">
      Public client identifier.
    </ParamField>

    <CodeGroup tabs="JavaScript,Python">
      ```javascript
      const response = await fetch('https://api.example.com/v1/docs', {
        headers: {
          'X-API-Key': 'YOUR_API_KEY',
          'X-Client-ID': 'your-client-id-here'
        }
      });
      ```
      ```python
      import requests
      headers = {
          'X-API-Key': 'YOUR_API_KEY',
          'X-Client-ID': 'your-client-id-here'
      }
      response = requests.get('https://api.example.com/v1/docs', headers=headers)
      ```
    </CodeGroup>

  </Tab>
  <Tab title="OAuth 2.0" icon="shield">

    Integrate with external providers like Google or GitHub for federated login.

    Redirect users to `https://auth.example.com/oauth/authorize?client_id=YOUR_CLIENT_ID&redirect_uri=YOUR_REDIRECT_URI&scope=read write`.

    After authorization, exchange the code for tokens at `https://auth.example.com/oauth/token`.

  </Tab>
</Tabs>

## Role Assignments and Permissions

Define permissions with roles. Each role grants specific capabilities.

| Role     | View Docs | Edit Docs | Manage Team | Delete Docs |
|----------|-----------|-----------|-------------|-------------|
| Viewer   | ✅        | ❌        | ❌          | ❌          |
| Editor   | ✅        | ✅        | ❌          | ❌          |
| Admin    | ✅        | ✅        | ✅          | ✅          |

<Steps>
  <Step title="Assign Role" icon="user-check">

    Update a user's role via the dashboard or API.

    ```javascript
    await fetch(`https://api.example.com/v1/users/${userId}/role`, {
      method: 'PATCH',
      headers: {
        'Authorization': `Bearer ${accessToken}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ role: 'editor' })
    });
    ```

  </Step>
  <Step title="Verify Permissions" icon="check-circle">

    Test endpoint access with the assigned role.

  </Step>
</Steps>

## Team Invitations and Access Levels

Invite team members directly from your dashboard.

1. Navigate to `https://dashboard.example.com/team`.
2. Click "Invite Member".
3. Enter email and select role (Viewer, Editor, Admin).
4. Send invitation.

Invited users receive an email with a secure link to accept.

<Expandable title="Advanced: Bulk Invitations via API" default-open="false">

Use the bulk API for large teams.

```javascript
await fetch('https://api.example.com/v1/team/invitations/bulk', {
  method: 'POST',
  headers: { 'Authorization': `Bearer ${accessToken}` },
  body: JSON.stringify([
    { email: 'team1@example.com', role: 'editor' },
    { email: 'team2@example.com', role: 'viewer' }
  ])
});
```

</Expandable>

## Security Best Practices

Follow these guidelines to maintain a secure documentation space.

<Columns cols={2}>
  <Card title="Rotate Keys Regularly" icon="refresh-cw" horizontal>
    Generate new API keys every 90 days. Revoke old ones immediately.
  </Card>
  <Card title="Use Least Privilege" icon="shield">
    Assign the minimum role needed. Auditors get Viewer access only.
  </Card>
  <Card title="Enable 2FA" icon="lock">
    Require two-factor authentication for all Admin users.
  </Card>
  <Card title="Audit Logs" icon="activity">
    Review access logs at `https://dashboard.example.com/audit`.
  </Card>
</Columns>

<Callout kind="tip">
Monitor your audit logs weekly. Set up webhooks to `https://your-webhook-url.com/webhook` for real-time alerts on suspicious activity.
</Callout>
